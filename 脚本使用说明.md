## 🖼️ Logo显示问题修复

### fix_domain_logo_display.sh ⭐ 推荐（针对域名访问）

**用途**：修复域名访问时logo无法显示的问题（IP访问正常，域名访问不正常）。

**使用方法**：

```bash
# 1. 上传脚本到服务器
scp fix_domain_logo_display.sh root@47.100.238.50:/root/

# 2. SSH登录服务器
ssh root@47.100.238.50

# 3. 执行脚本
bash /root/fix_domain_logo_display.sh
```

**功能**：
- ✅ 确认logo文件存在
- ✅ 测试IP访问
- ✅ 测试域名HTTP访问
- ✅ 测试域名HTTPS访问
- ✅ 优化Nginx配置
- ✅ 生成浏览器缓存清除说明

**常见结果**：

如果显示：
```
✅ 服务器端配置正常！
问题原因：浏览器缓存
```

说明服务器配置没问题，只需要清除浏览器缓存即可。

**清除浏览器缓存方法**：

1. **Chrome浏览器**：
   - 按 `Ctrl+Shift+Delete` (Windows) 或 `Cmd+Shift+Delete` (Mac)
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

2. **强制刷新**：
   - Windows: `Ctrl+F5`
   - Mac: `Cmd+Shift+R`

3. **使用无痕模式测试**：
   - Chrome: `Ctrl+Shift+N`
   - Firefox: `Ctrl+Shift+P`

## ⚙️ Nginx配置

### optimize_nginx_for_certbot.sh

**用途**：优化Nginx配置以支持certbot自动续期。

**使用方法**：

```bash
ssh root@47.100.238.50
bash /root/optimize_nginx_for_certbot.sh
```

**功能**：
- 备份nginx配置
- 创建certbot验证目录
- 显示推荐的nginx配置

**推荐配置**：

```nginx
# HTTP服务器 - 支持certbot验证并重定向到HTTPS
server {
    listen 80;
    server_name sxylab.com www.sxylab.com;

    # certbot验证路径（不重定向）
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    # 其他路径重定向到HTTPS
    location / {
        return 301 https://$server_name$request_uri;
    }
}

# HTTPS服务器
server {
    listen 443 ssl http2;
    server_name sxylab.com www.sxylab.com;

    ssl_certificate /etc/letsencrypt/live/sxylab.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/sxylab.com/privkey.pem;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location /static/ {
        alias /root/crm/static/;
        expires 7d;
        add_header Cache-Control "public";
    }

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 🔧 Nginx配置文件

项目包含多个Nginx配置文件模板：

1. **nginx_sxylab.conf** - 基础配置
2. **nginx_sxylab_https完整版.conf** - HTTPS完整配置
3. **nginx_sxylab_简化版.conf** - 简化版配置
4. **nginx_完整配置_sxylab.conf** - 完整配置模板

**使用方法**：

```bash
# 上传配置文件到服务器
scp nginx_sxylab_https完整版.conf root@47.100.238.50:/etc/nginx/sites-available/sxylab.com

# SSH登录服务器
ssh root@47.100.238.50

# 启用配置
ln -s /etc/nginx/sites-available/sxylab.com /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重新加载
systemctl reload nginx
```

## ❓ 常见问题

### 1. Logo在IP访问时正常，域名访问时不显示

**原因**：浏览器缓存

**解决方案**：
```bash
# 1. 运行诊断脚本
ssh root@47.100.238.50
bash /root/fix_domain_logo_display.sh

# 2. 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete
# 选择"缓存的图片和文件"

# 3. 强制刷新页面
# Windows: Ctrl+F5
# Mac: Cmd+Shift+R
```

---

### 2. SSL证书即将过期

**解决方案**：
```bash
# 1. 检查自动续期配置
ssh root@47.100.238.50
bash /root/check_ssl_auto_renew.sh

# 2. 手动续期
certbot renew

# 3. 如果自动续期未配置，运行配置脚本
bash /root/一键配置SSL自动续期.sh
```

---

### 3. 浏览器显示"不安全"

**原因**：HTTP没有重定向到HTTPS

**解决方案**：
```bash
# 1. 检查nginx配置
ssh root@47.100.238.50
grep -A 5 "server_name sxylab.com" /etc/nginx/nginx.conf

# 2. 确保HTTP服务器块包含重定向
# location / {
#     return 301 https://$server_name$request_uri;
# }

# 3. 重新加载nginx
nginx -t && systemctl reload nginx
```

---

### 4. 静态文件无法访问（404错误）

**解决方案**：
```bash
# 1. 运行logo修复脚本
ssh root@47.100.238.50
bash /root/fix_logo_display.sh

# 2. 检查文件是否存在
ls -la /root/crm/static/images/

# 3. 检查nginx配置
grep -A 3 "location /static/" /etc/nginx/nginx.conf

# 4. 确保配置正确
# location /static/ {
#     alias /root/crm/static/;  # 末尾必须有 /
# }
```
