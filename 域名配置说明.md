# sxylab.com 域名配置说明

## 📋 配置步骤

### 1. 上传Nginx配置文件到服务器

```bash
# 在本地执行（上传配置文件）
scp nginx_sxylab.conf root@47.100.238.50:/etc/nginx/sites-available/sxylab.com
```

### 2. 在服务器上启用配置

```bash
# SSH登录服务器
ssh root@47.100.238.50

# 创建软链接启用站点
ln -s /etc/nginx/sites-available/sxylab.com /etc/nginx/sites-enabled/

# 测试Nginx配置是否正确
nginx -t

# 如果测试通过，重新加载Nginx
systemctl reload nginx
```

### 3. 确保CRM服务正在运行

```bash
# 检查CRM服务状态
systemctl status crm

# 如果未运行，启动服务
systemctl start crm

# 查看服务是否监听5000端口
netstat -tlnp | grep 5000
# 或
ss -tlnp | grep 5000
```

### 4. 配置域名DNS解析

在您的域名注册商（如阿里云、腾讯云等）的DNS管理面板中添加以下记录：

```
类型: A
主机记录: @
记录值: 47.100.238.50
TTL: 600

类型: A
主机记录: www
记录值: 47.100.238.50
TTL: 600
```

### 5. 验证配置

```bash
# 等待DNS生效（通常5-10分钟）
# 测试域名是否可以访问
curl -I http://sxylab.com
curl -I http://www.sxylab.com

# 应该返回200状态码
```

## 🔒 配置HTTPS（推荐）

### 使用Let's Encrypt免费SSL证书

```bash
# 1. 安装certbot
apt update
apt install certbot python3-certbot-nginx -y

# 2. 自动获取并配置SSL证书
certbot --nginx -d sxylab.com -d www.sxylab.com

# 3. 按照提示输入邮箱并同意服务条款

# 4. 选择是否重定向HTTP到HTTPS（推荐选择2 - Redirect）

# 5. 测试自动续期
certbot renew --dry-run
```

certbot会自动修改Nginx配置文件，添加SSL配置并设置HTTP到HTTPS的重定向。

## 🔍 故障排查

### 问题1：域名无法访问

```bash
# 检查DNS是否生效
ping sxylab.com
nslookup sxylab.com

# 检查Nginx是否运行
systemctl status nginx

# 检查Nginx错误日志
tail -50 /var/log/nginx/sxylab_error.log

# 检查CRM服务是否运行
systemctl status crm
```

### 问题2：502 Bad Gateway

```bash
# 检查CRM服务是否正在运行
systemctl status crm

# 检查5000端口是否被监听
netstat -tlnp | grep 5000

# 查看CRM服务日志
journalctl -u crm -n 50

# 重启CRM服务
systemctl restart crm
```

### 问题3：静态文件无法加载

```bash
# 检查静态文件目录权限
ls -la /root/crm/static/

# 确保Nginx有读取权限
chmod -R 755 /root/crm/static/

# 检查Nginx配置中的路径是否正确
nginx -T | grep static
```

### 问题4：上传文件失败

```bash
# 检查上传目录是否存在
ls -la /root/crm/static/uploads/

# 创建上传目录（如果不存在）
mkdir -p /root/crm/static/uploads/course_records
mkdir -p /root/crm/static/uploads/award_certificates
mkdir -p /root/crm/static/uploads/teacher_images

# 设置权限
chmod -R 755 /root/crm/static/uploads/
chown -R www-data:www-data /root/crm/static/uploads/
```

## 📝 配置文件说明

### 关键配置项

1. **反向代理**
   - `proxy_pass http://127.0.0.1:5000` - 转发请求到Gunicorn服务
   - 端口5000必须与CRM服务监听的端口一致

2. **静态文件**
   - `/static/` - CSS、JS、图片等静态资源
   - `/static/uploads/` - 用户上传的文件

3. **上传限制**
   - `client_max_body_size 10M` - 限制上传文件大小为10MB
   - 可根据需要调整

4. **超时设置**
   - `proxy_read_timeout 60s` - 代理读取超时60秒
   - 如果有长时间运行的请求，可以增加此值

## 🔄 回滚方案

如果配置出现问题，可以快速回滚：

```bash
# 禁用新配置
rm /etc/nginx/sites-enabled/sxylab.com

# 重新加载Nginx
systemctl reload nginx

# 删除配置文件（可选）
rm /etc/nginx/sites-available/sxylab.com
```
